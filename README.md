# Cyvl Hackathon — Somerville Rain-Risk

Interactive web map **+** CAD pipeline for stormwater/ponding risk in **East Somerville / Inner Belt**.
Cyvl infrastructure (catch basins, manholes, pavement-PCI) is draped on a **LiDAR** terrain surface,
with an elevation-aware ponding-risk ranking.

## Web app (repo root) — deploys on Vercel
Next.js + Leaflet dark map of the data.
- `app/` + `components/RainRiskMap.jsx` — the map UI (client-only Leaflet, dark CARTO basemap).
- `public/data/*.geojson` — the layers (generated by `pipeline/prepare_web_data.py`).
- **Local:** `npm install && npm run dev` → http://localhost:3000
- **Deploy:** pushing to this repo auto-builds on Vercel (Next.js is auto-detected at the repo root).

Layers: ponding-risk ranking (color = priority), pavement (color = PCI), catch basins + manholes
(click a basin for its Cyvl street photo), terrain contours (toggle). A Top-10 risk sidebar flies to
each location on click.

## Pipeline (`pipeline/`)
- `build_dtm.py` — LiDAR cloud (EPSG:32619 / UTM 19N) → 1 m bare-earth DTM + depression grid.
- `build_dxf.py` — Cyvl layers → reproject → sample Z (LiDAR DTM + USGS EPQS cross-check) →
  elevation-aware ponding ranking → georeferenced 3-D **DXF** for Autodesk Civil 3D
  (`pipeline/output/somerville_rainrisk.dxf`) + companion GeoJSON/CSV.
- `prepare_web_data.py` — emits the web app's `public/data/*.geojson` (WGS84).
- `pipeline/output/README_DXF.txt` — Civil 3D import notes + the ranking formula.

## CRS & coverage notes
Pipeline/DXF outputs are **EPSG:32619** (UTM 19N, meters) to match the LiDAR; the web GeoJSON is WGS84.
Inside the LiDAR footprint Cyvl carries catch basins / manholes / pavement (not ramps/sidewalks/
markings — those cover central/west Somerville). The ranking is an **elevation-aware screen**, not a
hydraulic simulation — the full flood model runs in Autodesk on the LiDAR surface.
